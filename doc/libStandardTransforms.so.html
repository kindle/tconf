partial_update

Update string type FIDs based on partial update logic. If the commands are implemented to base string, 0 will be returned. Otherwise error code will be returned.

Example

tags: conversion, core, string, replace

partial_update ( errorCode, commands, base_value, ranges, option, result ) : variant
  • [out] errorCode : int64
    The error code
  • [in_out] commands : ascii_string
    The partial update command string. It contain the partial update rule. The repeat command will be replaced with insert command because it is not supported by downstream.
  • [in] base_value : variant
    The string value which the partial update command applying on.
  • [in] ranges : ascii_string
    The data range can be modified by partial update. The format should be N:M:L. N and M stand for the range can be modifed by partial update command. L stands for the max length of the input string. Example: 0:50:60 (Partial update command can update chars from 0 to 50 and the max length is 60)
  • [in] option : ascii_string
    The option used to control the partial update rule. 0 is used as default value. 0 is for No sepcial type. 1 is for DN60 Market field type A1. 2 is for DN60 Market field type A2
  • [out] result : ascii_string
    The result string after applying command to base string value.

tags: conversion, core, string, replace

generate_partial_update ( original_string, pos, update_substring, mask ) : variant
  • [in_out] original_string : ascii_string
    String to input
  • [in] pos : uint64
    The Position of the first character to be overwirtten by update_substring
  • [in] update_substring : ascii_string
    The substring that overwrites original_string with offset pos

    Mask for generating a partial update, if "", only content diff will be sent.
